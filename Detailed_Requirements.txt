2. Business Problems The existing boiler seller have the following problems: Weak customer management and market competitiveness: Facing the homogenized competition in industrial product trade and the upgrading of customer demands, companies have obvious shortcomings in customer service and market expansion. Low customer response efficiency: Customer demand records rely on manual registration, and customized equipment inquiries require repeated information transmission, with a long processing cycle, far exceeding the response speed of digital enterprises. The lack of digital marketing: Against the backdrop of e-commerce platforms and self-media marketing becoming mainstream, the company still relies on traditional offline promotion, making it difficult to reach the younger generation of equipment purchasing decision-makers. 3. Objectives Build an AI-enabled vertical trading platform for second-hand boilers to fill the gap in digital transactions in this niche field. Address the core pain points of information asymmetry, inefficient matching, and difficulty in ensuring credit between buyers and sellers of second-hand boilers. Verify the commercial feasibility of "AI technology + vertical industry" and form a replicable trading model for second-hand bulk commodities. Complete the full process of platform from design, development to testing on time to meet the requirements of the course project milestones. 4. Justification This project is worthwhile because it addresses an important problem in precision demand-supply matching for second-hand boiler transactions. (see subsection 8.1, 8.2) It is innovative as it is the first AI-integrated platform in large-machine trading in China. 10.3.3 AI Module AI Search: Support users to search for second-hand boiler related posts in natural language, solving the pain point of inaccurate user demand description; Interaction Logic: Users input natural language needs (e.g., "Purchasing a 10-ton second-hand steam boiler used for less than 3 years"), the system parses through AI, accurately matches relevant posts and returns search results, supporting further filtering by users; AI Customer Service: Provide 24/7 intelligent consulting services for users, answer common questions related to platform usage and transactions, and can transfer to manual consulting according to user needs. 10.3.4 Chat Module Function Description: Realize real-time chat communication between buyers and sellers, provide support for the preliminary docking of offline transactions, and assist AI customer service in completing consulting services. Interaction Logic: After users click the "Contact Seller" button, they will automatically enter the chat interface, where they can send text and pictures (such as actual boiler photos) and view historical chat records; AI customer service can access the current chat session to answer common questions for users. 10.3.5 Credit Module Objective: Establish a user credit evaluation system, evaluate user credit based on multi-dimensional parameters, and ensure the authenticity of platform information and transaction security. Function Description: Users can view the credit score and credit details of other users on their personal information pages Evaluation Mechanism: Following the completion of offline transactions, buyers and sellers may rate each other. These ratings will directly impact both parties’ credit scores. The system uses a 100-point credit score calculated via fixed rules, with no machine learning involved. Credit Scoring Criteria (Full mark: 100) Deposit Payment: 20 points - Full points granted upon successful guarantee deposit payment.

13. System Implementation 13.1 Technical Architecture 
 Figure 39 This project adopts a front-end and back-end separated architecture, which is designed to be scalable, maintainable and practical, fully meeting the needs of long-term iteration and production application. The overall architecture follows a clear layered design, with the user layer, gateway layer, application layer and data layer closely connected, and the technical selection is strictly based on the finalized technology stack without any redundant components. The user layer of the project is built with Node.js as the running environment and Vue.js as the core front-end framework, which is lightweight and easy to maintain, and can quickly respond to user operations while ensuring good interactivity. The gateway layer is implemented by Nginx, which serves as the unified entry point for all requests, undertaking functions such as request forwarding, load balancing and static resource management, effectively hiding the internal service details and ensuring the security and stability of the system. The application layer is developed based on Spring Boot and Spring MVC, which provides a mature and stable development environment. It uses JWT for stateless authentication to ensure the security of user access, and realizes real-time communication between buyers through WebSocket, while abandoning all unnecessary components that are not related to the core business. The data layer takes MySQL as the main relational database to store core business data such as user information, post content and transaction records, and uses Redis for high-performance caching to improve the response speed of the system. At the same time, it relies on MyBatis for ORM mapping and PageHelper for efficient paging query, which
